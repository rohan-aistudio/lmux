# lmux — Language Model Multiplexer

> Ollama-style CLI for dynamic LLM loading with llama.cpp + Open WebUI.  
> Pull models. Swap on demand. VRAM managed automatically.

## Architecture

```
lmux/
├── lmux.py           ← CLI engine (all platforms)
├── docker-compose.yml← generated by lmux init
├── config.yaml       ← auto-managed (do not edit)
├── registry.json     ← model metadata store
├── .env              ← HF token + settings
├── .venv/            ← isolated Python environment (uv)
├── models/           ← GGUF files (or custom path)
├── install.sh        ← Linux / macOS / WSL installer
├── install.ps1       ← Windows installer
└── uninstall.sh      ← Clean removal script
```

**Stack:**
- `llama-swap` → dynamic model router, VRAM-aware LRU eviction
- `llama-server` → spawned per model, exposes `/metrics` for token stats
- `Open WebUI` → chat UI with green status indicator, model selector

## Install

**One-liner (Linux / macOS / WSL):**
```bash
curl -sSL https://raw.githubusercontent.com/rohan-aistudio/lmux/main/install.sh | sh
```

The installer will:
1. Install **uv** if not present (Python package manager)
2. Clone lmux to `~/.lmux`
3. Create an **isolated virtual environment** (python ≥3.12) — no system Python conflicts
4. Install dependencies (`huggingface_hub`, `pyyaml`) inside the venv
5. Detect GPU and install NVIDIA Container Toolkit if needed
6. Start the Docker stack

**or clone and install manually:**
```bash
git clone https://github.com/rohan-aistudio/lmux.git ~/.lmux
cd ~/.lmux && bash install.sh
source ~/.bashrc   # or ~/.zshrc
```

**Windows (PowerShell as Admin):**
```powershell
powershell -ExecutionPolicy Bypass -File install.ps1
. $PROFILE
```

## Uninstall

**Linux / macOS:**
```bash
bash ~/.lmux/uninstall.sh
```

**Remove everything including downloaded models:**
```bash
bash ~/.lmux/uninstall.sh --delete-models
```

**Skip confirmation:**
```bash
bash ~/.lmux/uninstall.sh --delete-models --yes
```

This removes:
- Docker containers (`lmux-engine`, `lmux-ui`) and volumes
- Python virtual environment (`.venv`)
- Shell aliases from `.bashrc` / `.zshrc` / PowerShell profile
- lmux installation directory (`~/.lmux`)
- Downloaded models (only with `--delete-models`)

## Commands

| Command | What it does |
|---|---|
| `lmux init` | Detect GPU/OS, write compose, start stack, add alias |
| `lmux init --models-path /mnt/d/models` | Store models on a different drive |
| `lmux pull <url\|repo>` | Download GGUF from HuggingFace |
| `lmux ls` | List models with VRAM estimates and live status |
| `lmux info <name>` | Show metadata, VRAM headroom |
| `lmux down <name>` | Remove from config, **keep file on disk** |
| `lmux up <name>` | Add to config from model file on disk |
| `lmux rm <name>` | Remove model from config |
| `lmux rm <name> --delete-file` | Remove + delete GGUF from disk |
| `lmux status` | Live stack health + loaded models + live VRAM |
| `lmux stats` | Token throughput, KV cache usage, tok/s |
| `lmux run <name> "prompt"` | One-shot CLI inference |
| `lmux reload` | Resync config, restart engine |
| `lmux start` | Start Docker stack |
| `lmux stop` | Stop stack, free all VRAM |

## Usage Examples

```bash
# Pull by full URL
lmux pull https://huggingface.co/bartowski/Meta-Llama-3-8B-Instruct-GGUF/blob/main/Meta-Llama-3-8B-Instruct-Q4_K_M.gguf

# Pull by shorthand + quant filter
lmux pull bartowski/Meta-Llama-3-8B-Instruct-GGUF --quant Q4_K_M

# Pull with custom name shown in Open WebUI
lmux pull bartowski/Meta-Llama-3-8B-Instruct-GGUF --quant Q4_K_M --name llama3-8b

# List everything
lmux ls

# Get VRAM info on a specific model
lmux info llama3-8b

# Remove from config, keep file on disk
lmux down llama3-8b

# Add to config, from model file on disk
lmux up llama3-8b

# Remove from config AND delete the GGUF
lmux rm llama3-8b --delete-file

# Force resync if you edited registry.json manually
lmux reload

# Force CPU (even if GPU present)
CUDA=0 lmux pull bartowski/Phi-3-mini-4k-instruct-GGUF --quant Q4_K_M
lmux pull ... --cpu

# Quick inference from terminal
lmux run llama3-8b "Explain VRAM paging in 2 sentences"

# Token stats for loaded model
lmux stats

# Live status with GPU VRAM bar
lmux status
```

### Custom Model Storage Path

Store models on a different drive (useful for NVMe, external drives, NAS):

```bash
# During init — models go to /mnt/nvme/lmux-models
lmux init --models-path /mnt/nvme/lmux-models

# Subsequent pulls auto-download to the configured path
lmux pull bartowski/Meta-Llama-3-8B-Instruct-GGUF --quant Q4_K_M
```

The path is persisted in `registry.json` and mounted as a Docker volume in `docker-compose.yml`.

## How Tab-Based VRAM Swapping Works

| Event | What lmux does |
|---|---|
| Tab A → select Model X | llama-swap spawns llama-server on X's port, loads to GPU |
| Tab B → select Model Y | If VRAM full: X evicted (LRU), Y loaded |
| Both tabs active | Y waits for X to finish current request, then swaps |
| No activity for 5 min | Model process exits, VRAM fully freed |
| New tab, new model | Clean spawn into freed VRAM |

`GGML_CUDA_ENABLE_UNIFIED_MEMORY=1` — if a model exceeds GPU VRAM (8GB on RTX 4090),  
overflow spills into your 32GB system RAM. Inference continues, slightly slower on overflow layers.

## Programmatic Access (OpenAI-compatible)

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:11435/v1",
    api_key="lmux-local"
)

response = client.chat.completions.create(
    model="llama3-8b",       # any name from: lmux ls
    messages=[{"role": "user", "content": "Hello"}],
    stream=True,
)

for chunk in response:
    print(chunk.choices[0].delta.content or "", end="")
```

```bash
# curl
curl http://localhost:11435/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"llama3-8b","messages":[{"role":"user","content":"Hi"}]}'
```

## Open WebUI Green Status Indicator

`ENABLE_OLLAMA_API=true` makes Open WebUI treat lmux as an Ollama backend.  
This activates the green dot connection indicator and the active model status panel  
without requiring Ollama to be installed. Users can see which models are currently  
loaded and trigger unloads directly from the Open WebUI interface.

## Live Stats (CLI)

> **Note:** Live stats (VRAM usage, tok/s, KV cache) are available via the **CLI only** (`lmux status` / `lmux stats`).  
> Open WebUI shows the green connection indicator and model selector, but does not natively display token throughput or VRAM consumption.  
> For a full in-browser stats dashboard, consider running llama.cpp's built-in svelte-webui alongside Open WebUI.

`lmux status` shows a real-time GPU VRAM bar with usage percentage, free memory,  
GPU utilization, and temperature:

```
  → Live GPU Stats:
    VRAM:  [████████████████░░░░░░░░░░░░░░] 4820MB / 8192MB (59%)
    Free:  3372MB  |  GPU Load: 34%  |  Temp: 62°C
```

`lmux stats` shows per-model token throughput from llama-server's Prometheus `/metrics`:
- Prompt tokens total
- Generated tokens total
- Generation speed (tok/s)
- KV cache usage
- Active/queued requests
